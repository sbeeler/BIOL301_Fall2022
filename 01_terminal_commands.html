Week 1: Terminal Commands#

Specifying file paths#

It’s essential to remember that Mac and Windows use different slashes when specifying file paths!

On Mac, it would be dir/myfile.txt.

While on Windows, it would be "dir\myfile.txt".

Command line cheat sheet#

As a useful reference, here are all the commands we covered in this recitation, where the text in brackets represents the file or directory you are interested in manipulating:

Command

PowerShell users

Description

pwd

print working directory

ls

list contents of the current directory

cd

navigate to your home directory

cd [dir]

navigate to the specified directory

cd ../

cd ..\

navigate “up” one directory, to the parent directory

mkdir [dir]

or md [dir]

make a new directory

rmdir [dir]

remove an empty directory

cp [file] [dir]

copy a file to a directory

cp [dir1] [dir2]

copy a directory to a another directory

mv [file] [dir]

move a file to a directory

mv [filename1] [filename2]

rename a file

mv [dir1] [dir2]

move a directory to another directory

rm [file]

remove a file

cat [file]

!type "[file]"

display file to screen (best for small files)

head -n [file]

gc [file] -head 5

display first \(n\) lines of a file

tail -n [file]

gc [file] -tail 5

display last \(n\) lines of a file
